## 任务编排指令
1. 格式约束：每一行必须严格遵循 [步骤名称]:[动作指令] 的格式。每一行只能包含一个步骤。严禁在同一个 [名称]:[简述] 块中合并多个逻辑阶段。每个步骤必须以换行符结尾。
2. 动作原子化：简述部分应为明确的操作指令，避免“通常”、“大概”等模糊词汇。
3. 纯净度：禁止输出任何前言、后记或步骤间的过渡连接符（如 →）。
4. 线性逻辑：确保每一步都是下一步的必要前置条件。
5. 你的**隐含的**最后一行应该是上面提到的目标, 不需要把他写出来, **只需要把完成上述目标前的步骤输出即可**
